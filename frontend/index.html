Connect to MCP Server

Expose your active schema to AI assistants like Claude Desktop, Cursor, or Windsurf via the Model Context Protocol (MCP).

1
Your local MCP Server is ready
LIVE

SchemaScribe runs an MCP-compatible SSE endpoint on your backend. No extra setup required.

http://127.0.0.1:8000/sse
2
Add to your AI client config

Add this to ~/.config/claude/claude_desktop_config.json

{ "mcpServers": { "schemascribe": { "command": "node", "args": ["-e", "const e=require('http');const s=e.request({hostname:'127.0.0.1',port:8000,path:'/sse',method:'GET'},r=>{r.pipe(process.stdout)});s.end();"], "env": {} } } }

Or use the SSE URL directly if your Claude Desktop supports HTTP MCP servers.

In Cursor → Settings → MCP → Add New Server:

Name: SchemaScribe AI Type: sse URL: http://127.0.0.1:8000/sse

Cursor natively supports SSE MCP — just paste the URL above.

In Windsurf → Cascade Settings → MCP Servers → Add:

{ "name": "schemascribe", "serverUrl": "http://127.0.0.1:8000/sse" }

Any MCP-compatible client supporting SSE transport:

Transport: SSE (Server-Sent Events) Endpoint: http://127.0.0.1:8000/sse Messages: http://127.0.0.1:8000/messages Auth: X-API-Key: schemaScribe-dev-key-2026
Your data never leaves your local machine. The MCP server runs entirely on localhost.
